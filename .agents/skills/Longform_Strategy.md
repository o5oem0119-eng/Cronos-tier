# 📺 Longform Strategy v2.0 — 크로노스 엔진 롱폼 제작 전략

> **이 문서는 프로젝트의 대본 생성 품질을 결정하는 최상위 전략 문서입니다.**
> TierZoo의 실제 대본 20편+ 분석, 벤치마킹 가이드, wiki 28개 유튜브 제작 가이드라인을 통합하여 작성되었습니다.
> 모든 에이전트는 대본 작성 시 이 문서를 `TierZoo_Styling_Guide` 스킬과 함께 반드시 참조해야 합니다.

## 0. 🧭 설계 철학 (Design Philosophy)

### TierZoo 성공 공식의 핵심 3원칙

1. **세계관의 일관성 (World Consistency):** 지구를 'Outside'라는 MMORPG로 설정한 것처럼, 우리는 역사를 **'Joseon Server', 'Goryeo Expansion'** 등의 게임 세계로 철저히 롤플레잉한다. 한 순간이라도 게임 세계관에서 이탈하면 몰입이 깨진다.
2. **분석의 객관성 (Analytical Objectivity):** 역사의 비극을 감상적으로 서술하지 않는다. e스포츠 해설자처럼 냉정하게 **"왜 이 빌드가 실패했는가"**를 시스템적으로 분석한다. 괴리감에서 오는 유머가 우리 채널의 정체성이다.
3. **정보의 에듀테인먼트 (Edutainment):** 시청자가 10분 동안 웃고 즐기며 게임 공략 영상을 본 것 같지만, 다 보고 나면 수준 높은 역사 지식을 얻게 만든다.

### 기존 전략 vs 신규 전략 핵심 변경점

| 기존 (v1.0) | 신규 (v2.0) | 변경 이유 |
| :--- | :--- | :--- |
| 5단계 피상적 구조 | **TierZoo 공식 기반 7단계 정밀 구조** | 실제 대본 분석 결과 반영 |
| 3분/6분 2개 브레이크포인트 | **60~90초 간격 연속 리텐션 방어** | 유튜브 리텐션 심리학 적용 |
| 게임 UI 위주 비주얼 | **투 칼럼(AV) 스크립트 + 페이싱 지시어** | 영상 자동화 파이프라인 호환 |
| 단순 용어 치환 | **다층적 게임 메커니즘 비유 체계** | TierZoo 벤치마킹 심층 분석 |

## 1. 🎯 서사 구조: TierZoo 공식 기반 7단계 (The Chronos Formula)

10분 영상 기준. 분량은 주제의 복잡도에 따라 ±30초 유연 적용.

### Stage 1: 도발적 선언 (The Hook) — `[0:00 ~ 0:10]`

> **TierZoo 공식:** "Cheetahs are low tier trash." 한 문장으로 통념을 깨부순다.

- **목적:** 스크롤 멈춤. 3초 이내에 인지적 충격(Cognitive Dissonance) 유발.
- **역사 채널 적용:**
  - ❌ "오늘은 단종에 대해 알아보겠습니다." (설명형 — 즉시 이탈)
  - ✅ "조선 서버 역대 최고의 적통성 빌드가 12세에 강제 로그아웃 당한 이유? 가디언 슬롯이 비었기 때문입니다." (도발형)
- **필수 요소:**
  - 핵심 결론/반전을 선공개 (결과 먼저, 맥락은 나중에)
  - 구어체, 짧은 호흡 (50단어 이내)
  - `[Visual: 캐릭터 카드 UI 급격 줌인 + 경고등 점멸]`
  - `[SFX: 긴장감 있는 전자음 또는 게임 경고음]`

### Stage 2: 메타 정의 & 로드맵 (The Setup) — `[0:10 ~ 1:30]`

> **TierZoo 공식:** "이 서버의 메타가 무엇인지" 정의 + "오늘 이 빌드의 스탯, 능력, 상성, 최종 티어를 분석합니다" 로드맵 제시.

- **목적:** 시청자에게 "이 영상은 체계적 분석이다"라는 신뢰감 부여 + 끝까지 볼 이유 생성.
- **역사 채널 적용:**
  - 해당 시대가 어떤 '메타'인지 1~2문장으로 정의
  - 예: "조선 서버는 적통성이 없으면 바로 강퇴당하는 하드코어 적통성 메타입니다."
  - 분석 대상(인물/사건)을 '빌드'로 공식 소개
  - **로드맵 선언:** "오늘 이 빌드의 상속 스탯, 가디언 시스템, 카운터 픽 상성, 그리고 최종 티어를 분석해 보겠습니다."
- **필수 요소:**
  - 해당 시대의 '패치 버전' 또는 '확장팩' 넘버링 (예: `Joseon Server v1.4`)
  - 30초 부근에서 `[PATTERN INTERRUPT: 타이틀 카드 팝업 + 음악 전환]` 필수
  - 첫 번째 `[OPEN LOOP]` 삽입 — "하지만 이 완벽한 빌드에 치명적인 결함이 하나 있었습니다. 잠시 후에 알려드리죠."

### Stage 3: 기본 스탯 분석 (Base Stats) — `[1:30 ~ 3:30]`

> **TierZoo 공식:** 육각형 스탯(HP, DEF, PWR, MOB, INT, STL)으로 캐릭터의 기본 역량을 수치화.

- **목적:** 분석 대상의 강점/약점을 직관적으로 전달. 시청자가 "이 캐릭터, 생각보다 약하네?" 또는 "스탯은 사기인데?" 하는 반응 유도.
- **역사 채널 적용 (스탯 치환 기준):**

| 게임 스탯 | 역사적 의미 | 평가 기준 |
| :--- | :--- | :--- |
| **CHA (적통성/카리스마)** | 왕위 계승의 정당성, 명분 | 혈통, 왕비 출신, 선왕의 유지 |
| **INT (지능/전략)** | 정치적 수완, 외교력, 전략 설계 | 실제 정책, 모략, 정보망 |
| **STR (물리력/군사력)** | 직접적 무력 동원 능력 | 사병, 군권 장악력, 호위 무사 |
| **HP (생존력/체력)** | 물리적 건강, 정치적 방어력 | 나이, 건강 상태, 보호막 유무 |
| **DEF (방어력)** | 제도적 보호 장치 (수렴청정, 보정대신) | 가디언 NPC 수, 제도적 버프 |
| **LUK (행운)** | 시대적 운, 우연적 사건의 영향 | 자연재해, 외세 개입 등 |

- **필수 요소:**
  - `[Visual: 육각형 스탯 그래프 또는 캐릭터 스탯 카드 UI]`
  - 다른 인물과의 직접 비교 (TierZoo가 다이어울프를 회색늑대와 비교하듯)
  - 스탯의 **트레이드오프** 반드시 언급 — "적통성 올인 빌드의 대가로 물리력이 바닥입니다."
  - 3분 부근 `[PATTERN INTERRUPT]` + 두 번째 `[OPEN LOOP]`

### Stage 4: 특수 능력 & 시스템 글리치 (Abilities & Exploits) — `[3:30 ~ 5:30]`

> **TierZoo 공식:** 기본 스탯 외의 고유 스킬/패시브/익스플로잇 분석. 거미의 거미줄, 박쥐의 반향정위처럼 "게임 밸런스를 파괴하는" 능력.

- **목적:** 주인공 또는 적의 고유 능력이 어떻게 스탯의 한계를 보완/파괴하는지 분석.
- **역사 채널 적용:**
  - **패시브 버프:** 선왕의 유지, 명나라의 공인, 신료들의 충성 맹세 등
  - **액티브 스킬:** 밀지 작성, 군사 동원, 사신 파견, 밀정 운용
  - **시스템 익스플로잇:** 정변, 옥새 탈취, 선위 강요 등 규칙 파괴 행위
  - **디버프/글리치:** 왕의 어린 나이, 수렴청정 불가, 후견인 부재 등
- **필수 요소:**
  - `[Visual: 패치 공지 / SYSTEM LOG 형식의 텍스트 오버레이]`
  - 예시 포맷: `[SYSTEM LOG] 메인 탱커 '문종' 유닛이 서버에서 제거되었습니다.`
  - 적의 능력을 소개할 때 긴장감 극대화 — 이 구간이 **감정의 첫 번째 하락 지점**

### Stage 5: 상성 분석 & 보스전 (Matchups & The Clash) — `[5:30 ~ 8:00]`

> **TierZoo 공식:** 1대1 상성표, 카운터 플레이, 팀 콤보 분석. 가장 분량이 긴 핵심 파트.

- **목적:** 대립 구도의 클라이맥스. 왜 주인공이 졌는지 / 이겼는지를 시스템적으로 증명.
- **역사 채널 적용:**
  - **1v1 스탯 비교표** (마크다운 테이블)로 양측의 역량 시각화
  - **카운터 플레이 분석:** 적이 어떤 전략으로 주인공의 강점을 무효화했는지
  - **팀 콤보:** 길드원(조력자) 영입, 밀약, 연합 세력 형성 과정
  - **서버(환경) 메타의 영향:** 당시 시대적 조건이 어느 쪽에 유리했는지
- **필수 요소:**
  - `[Visual: 실시간 스탯 바 대칭 비교 + 결과 표시]`
  - 이 구간에서 **최대 긴장 지점(Peak Tension)** 도달
  - 6분 부근 `[PATTERN INTERRUPT: 화면 붉은색 필터 + 경고음]`
  - **사건 발생을 `[SYSTEM LOG]` 형식으로 실시간 중계**
  - 세 번째 `[OPEN LOOP]` — "하지만 진짜 잔인한 건 이 다음입니다."
  - **이 단계에 영상의 70% 클라이맥스 배치** (Peak-End Rule)

### Stage 6: 엔드게임 & 결말 (Endgame) — `[8:00 ~ 9:00]`

> **TierZoo 공식:** 분석 대상의 최종 운명. 밴(Ban), 계정 삭제, 서비스 종료, 또는 복구 패치.

- **목적:** 클라이맥스의 여파를 정리하고 감정의 마무리.
- **역사 채널 적용:**
  - 주인공의 최종 상태를 간결하게 서술 (유배, 사약, 복위 등)
  - **사후 복구 패치:** 후대 왕에 의한 복권, 추증, 사당 건립 등
  - 감정적이지 않되, **시스템적 냉소** 속에 여운이 남도록 — "조선 서버 역사상 가장 부당한 밴 처리였죠."
- **필수 요소:**
  - `[Visual: 계정 삭제 / 서비스 종료 UI]`
  - 마지막 `[SYSTEM LOG]` — 최종 처리 기록

### Stage 7: 최종 티어 & 시그니처 아웃트로 (Tier Placement & Outro) — `[9:00 ~ 10:00]`

> **TierZoo 공식:** 스탯/능력/상성을 종합한 최종 티어 부여 + "Good luck out there." 시그니처.

- **목적:** 영상의 논리적 결론. 시청자의 토론/댓글 유도.
- **역사 채널 적용:**
  - **복합 티어 평가** (단일 등급이 아닌 다면 평가):
    - 능력치 계승도: S/A/B/C/D/F
    - 생존 효율성: S/A/B/C/D/F
    - 메타 영향력: S/A/B/C/D/F
    - 사후 유산도: S/A/B/C/D/F
  - **최종 종합 티어** 선언
  - **시그니처 아웃트로:** "다음엔 또 어떤 억울하게 밴 당한 캐릭터가 우리를 기다리고 있을까요? 기대하세요. 그리고 언제나 그렇듯, 이 서버에서 행운을 빕니다."
- **필수 요소:**
  - `[Visual: 티어 리스트 UI에 캐릭터 아이콘 배치 애니메이션]`
  - 끝남의 암시어 금지 ("결론적으로", "감사합니다" 등)
  - 다음 에피소드 티저로 연결 — `[CTA: 다음 영상 썸네일 팝업]`

## 2. ⚡ 리텐션 방어 시스템 (Retention Defense System)

### 2.1 연속 리텐션 방어 (60~90초 간격)

TierZoo 대본 분석 결과, 단순히 3분/6분에 브레이크를 거는 것으로는 부족합니다.
**60~90초마다** 아래 3가지 장치 중 최소 1개를 반드시 삽입하십시오.

| 장치 | 태그 | 설명 | 예시 |
| :--- | :--- | :--- | :--- |
| **오픈 루프** | `[OPEN LOOP]` | 미해결 궁금증을 던져 다음 섹션까지 시청 유도 | "하지만 진짜 무서운 건 이 다음에 벌어집니다." |
| **패턴 인터럽트** | `[PATTERN INTERRUPT]` | 시각/청각 변化로 주의력 리셋 | `[SFX: 게임 레벨업 효과음 + 화면 플래시]` |
| **감정 전환** | `[EMOTION FLIP]` | 긍정↔부정 감정의 급격한 전환 | "스탯은 완벽했습니다. 문제는, 이 서버에서 그게 전혀 의미가 없었다는 거죠." |

### 2.2 감정 곡선 설계 (Emotional Arc)

크로노스 엔진의 역사 에피소드는 **기본적으로 'Icarus(상승 후 하락)' 또는 'Oedipus(하락-상승-하락)'** 패턴을 따릅니다.

```
감정(Emotion)
    ▲
    │      ╱╲ Stage 3: 스탯 소개 (이 빌드 사기네!)
    │     ╱  ╲
    │    ╱    ╲ Stage 4: 시스템 글리치 발견 (어...?)
    │   ╱      ╲
    │  ╱        ╲ Stage 5: 보스전 패배 (으악)
    │ ╱          ╲
    │╱  Stage 2    ╲_____________ Stage 6: 엔드게임
    │                             ╲
    │  Stage 1: Hook (충격)        ╲ Stage 7: 사후 복구 (여운)
    └──────────────────────────────────▶ 시간(Time)
```

### 2.3 3-5초 규칙

시청자가 한 장면에 집중하는 시간은 **3~5초**입니다.
- 화자 중심의 정적 장면이 5초 이상 지속되지 않도록 시각적 변화를 삽입
- 3~4문장마다 `[Visual]` 지시어 필수
- 택 1: `[줌인]`, `[자료화면 교차]`, `[텍스트 팝업]`, `[스탯 바 변동]`, `[맵 이동]`

## 3. 🎮 다층적 게임 비유 체계 (Game Metaphor System)

### 3.1 세계관 치환 사전 (World-Building Dictionary)

단순한 단어 바꾸기가 아니라, **역사적 메커니즘을 게임 시스템으로 논리적 치환**합니다.

| 역사적 개념 | 게임적 치환 | 활용 맥락 |
| :--- | :--- | :--- |
| 조선/고려/국가 | 서버 (Server) | "조선 서버 v1.4 패치" |
| 시대/왕조 교체 | 확장팩 (Expansion) / 밸런스 패치 | "고려→조선 서버 마이그레이션" |
| 국왕/왕세자 | 서버 관리자 / 서브 관리자 | "관리자 권한 승계" |
| 정통성/혈통 | 적통성 스탯 (CHA/Legitimacy) | "적통성 가챠에서 SSS 등급 뽑음" |
| 즉위 | 관리자 로그인 / 권한 승계 | "리더 권한 강제 승계 완료" |
| 폐위/유배 | 계정 정지 / 맵 격리 / 서버 추방 | "영월 바이옴으로 격리 수용" |
| 사약/처형 | 계정 삭제 / 밴(Ban) / 강제 로그아웃 | "최종 삭제 스크립트 실행" |
| 복권/추증 | 복구 패치 / 롤백(Rollback) | "숙종의 복구 패치 v3.2" |
| 정변/쿠데타 | 시스템 해킹 / 익스플로잇 (Exploit) | "권한 탈취 익스플로잇 발동" |
| 전쟁/전투 | PVP / 레이드 / 보스전 | "살수대첩: 역대급 팀 와이프" |
| 외교/동맹 | 파티 플레이 / 길드 연합 | "명나라 서버와 크로스서버 동맹" |
| 모략/밀약 | 해킹 / 백도어 / 스크립트 실행 | "한명회의 살생부 스크립트" |
| 충신/조력자 | 가디언 NPC / 서포터 / 파티원 | "가디언 NPC 김종서 유닛" |
| 적/정적 | 카운터 픽 / 하드 카운터 / 메타 브레이커 | "수양: 전형적 파워 크립 빌드" |
| 자연재해/기근 | 환경 디버프 / 서버 버그 | "흉년 이벤트: 전 서버 식량 너프" |
| 법/제도 | 시스템 룰 / 서버 정책 | "수렴청정: 자동 방어 시스템" |
| 문화/학문 | 테크 트리 / 스킬 해금 | "한글: 커뮤니케이션 테크 트리 해금" |

### 3.2 전투 용어 활용 가이드

TierZoo처럼 **하드코어 게이머 은어**를 자연스럽게 섞어 사용합니다.

- **DPS / 크리티컬 히트:** 타격의 강도와 치명도
- **어그로 (Aggro):** 정치적 주목/견제를 끄는 행위
- **광역기 (AoE):** 광범위한 영향력 행사 (대규모 숙청 등)
- **스폰 킬 / 캠핑:** 세력 기반이 약한 대상을 집중 공격
- **버프 / 너프:** 특정 세력/인물의 강화/약화
- **파워 크립 (Power Creep):** 점차 강해지는 적의 위협
- **민맥싱 (Min-Maxing):** 특정 능력에 올인하고 나머지를 포기한 빌드
- **트레이드오프 (Trade-off):** 능력 간의 교환 관계
- **유지비 (Upkeep Cost):** 거대한 권력/영토를 유지하는 비용

### 3.3 비유의 심도 규칙 (Depth Rules)

> [!IMPORTANT]
> **단순 치환 금지.** 단어만 바꾸는 것은 식상합니다.
> 반드시 **메커니즘 수준의 비유**를 구사하십시오.

- ❌ 얕은 비유: "단종은 12세에 왕이 되었습니다. 게임으로 치면 뉴비(Newbie)죠."
- ✅ 깊은 비유: "만렙 아이템을 두른 채 필드 보스들만 가득한 PVP 구역에 홀로 던져진 셈입니다. 장비는 전설급인데 레벨이 12. 이건 그냥 아이템 사냥꾼들의 먹잇감이죠."

## 4. 🎬 비주얼 스크립트 포맷 (Visual Script Format)

### 4.1 투 칼럼(AV) 스크립트 기본 구조

모든 대본은 시각(VIDEO)과 청각(AUDIO) 요소를 **분리**하여 작성합니다.
이는 Remotion 자동 렌더링 파이프라인과의 직접적인 호환을 위한 것입니다.

```markdown
| VIDEO (시각 요소) | AUDIO (청각 요소) |
| :--- | :--- |
| **[0:00-0:05] Hook** | |
| `FAST ZOOM` 단종 캐릭터 카드. | (SFX: WARNING BEEP) |
| STR: 5 / LUK: 0 깜빡임 | NARRATOR: "스탯은 만렙인데 방어력이 0인 상태로..." |
| **[0:05-0:10] Setup** | |
| `FLASH` 화면 반전 + 타이틀 카드 | (MUSIC: 긴장감 있는 가야금 비트) |
| 'Joseon Server Season 1' 팝업 | NARRATOR: "조선 서버의 랭크전은 단 하나의 스탯으로..." |
```

### 4.2 필수 비주얼 지시어 태그

| 태그 | 용도 | 설명 |
| :--- | :--- | :--- |
| `[STAT CARD]` | 캐릭터 소개 | 육각형 스탯 그래프 + 빌드 프로필 |
| `[STAT COMPARE]` | 1v1 대결 | 양측 스탯 바 대칭 비교 |
| `[SYSTEM LOG]` | 사건 중계 | 시스템 메시지 형식의 텍스트 오버레이 |
| `[PATCH NOTE]` | 시대 변화 | 패치 공지 형식의 UI 팝업 |
| `[TIER CARD]` | 최종 평가 | 티어 리스트 UI + 아이콘 배치 |
| `[MAP MOVE]` | 장소 이동 | 서버 맵 위 이동 경로 표시 |
| `[HP BAR]` | 위기 표현 | HP 바 급격 감소 + 경고음 |
| `[FLASH]` | 전환 효과 | 화면 반전 + 장면 전환 |
| `[ZOOM]` | 강조 | 특정 요소 줌인/줌아웃 |

### 4.3 역사적 상황별 연출 장르 가이드

| 역사적 상황 | 권장 연출 장르 | 게임 문법 예시 |
| :--- | :--- | :--- |
| 대규모 전쟁 / 전투 | **RTS (전략 시뮬레이션)** | 유닛 배치, 전장 미니맵, 병력 수치 표시 |
| 정치 / 외교 / 밀약 | **4X / 대전략** | 세력 영향권 맵, 동맹 네트워크 도식 |
| 개인 대결 / 정변 | **격투 게임 / RPG** | 1v1 스탯 비교, VS 화면, HP 바 |
| 캐릭터 / 인물 분석 | **캐릭터 셀렉트** | Pick 창, 스탯 육각형, 장비 슬롯 |
| 시대 변화 / 개혁 | **업데이트 공지** | 패치 노트 UI, 변경사항 목록 |
| 유배 / 이동 | **오픈 월드** | 미니맵 경로, 바이옴 정보 창 |

## 5. 🗣️ 톤 & 보이스 가이드 (Tone & Voice Guide)

### 5.1 페르소나

- **역할:** 'Joseon Server' 전문 분석가 / e스포츠 역사 해설자
- **시점:** 철저하게 초월적 — "이 서버의 관리자들은 도대체 뭘 한 걸까요?"
- **태도:** 냉소적이되 객관적. 비극은 감상이 아니라 **시스템적 결함**으로 처리.

### 5.2 문체 규칙

| 규칙 | 설명 |
| :--- | :--- |
| **구어체 필수** | 12세도 이해할 수 있는 짧고 직관적인 말투. 논문체 절대 금지. |
| **1분당 130~150단어** | 페이싱 기준. 숏츠(60초)는 150단어 이내. |
| **호흡 짧게** | 한 문장은 2줄 이내. 긴 설명은 쪼개서 전달. |
| **금지어** | "안녕하세요", "시작하기에 앞서", "결론적으로", "감사합니다" |

### 5.3 시그니처 표현 (Signature Phrases)

TierZoo의 시그니처를 역사 채널에 맞게 변형:

- **"I hope you see what I'm getting at."** → "제가 무슨 말 하려는지 감이 오시죠?"
- **"Low tier trash."** → "하위 티어 쓰레기 빌드."
- **"Good luck out there."** → "이 서버에서 행운을 빕니다."
- **"The devs really dropped the ball."** → "이 서버 운영진은 대체 뭘 한 겁니까."
- **"This build may not look like much, but..."** → "이 빌드, 보기에 별거 없어 보이죠? 하지만..."

### 5.4 유머 패턴

| 패턴 | 설명 | 예시 |
| :--- | :--- | :--- |
| **괴리감 유머** | 비극을 시스템 결함으로 담담히 처리하는 데서 오는 웃음 | "물리적 보호막이 전부 오프라인. 이건 그냥 서비스 사고입니다." |
| **게임 커뮤니티 밈** | 게이머 커뮤니티에서 통용되는 밈 활용 | "적통성 가챠 SSS를 뽑았는데 보호자 슬롯이 비어있음 ㅋㅋ" |
| **냉소적 조언** | 시청자에게 플레이 조언하듯 냉정하게 조언 | "이 서버에서 이 빌드를 픽하려면, 가디언 NPC 최소 3기는 확보하세요." |

## 6. 📏 품질 체크리스트 (Quality Checklist)

대본 완성 후 아래 항목을 모두 점검합니다.

### 서사 구조
- [ ] 7단계 구조가 빠짐없이 적용되었는가?
- [ ] 첫 10초에 도발적 선언(Hook)이 있는가?
- [ ] 30초 이내에 로드맵이 제시되었는가?
- [ ] 최종 티어 평가가 다면적(4개 이상 항목)으로 이루어졌는가?

### 리텐션 방어
- [ ] 60~90초 간격으로 `[OPEN LOOP]` 또는 `[PATTERN INTERRUPT]`가 삽입되었는가?
- [ ] 영상의 70% 지점에 클라이맥스(Peak Tension)가 있는가?
- [ ] 끝남을 암시하는 단어 없이 자연스럽게 아웃트로로 연결되는가?

### 게임 비유
- [ ] 세계관 치환이 **메커니즘 수준**에서 이루어졌는가? (단순 단어 교체 ❌)
- [ ] `[SYSTEM LOG]`, `[STAT CARD]` 등 비주얼 태그가 적절히 삽입되었는가?
- [ ] 게이머 은어가 자연스럽게 녹아들었는가? (억지 삽입 ❌)

### 톤 & 문체
- [ ] 구어체로 작성되었는가? (논문체 ❌)
- [ ] 1분당 130~150단어 분량을 준수하는가?
- [ ] 금지어("안녕하세요", "감사합니다" 등)가 사용되지 않았는가?
- [ ] 감상적 신파 대신 시스템적 분석 톤이 유지되는가?

### 자동화 호환
- [ ] 투 칼럼(AV) 스크립트 형식으로 작성되었는가?
- [ ] 모든 비주얼 지시어에 대괄호 `[...]` 태그가 포함되었는가?
- [ ] 각 Stage별 타임라인이 명시되었는가?
