# 📜 Script Writing Rules: 단종의 비극 에피소드

이 문서는 `단종_비극_게임분석.md`의 스타일을 영상 대본으로 이행하기 위한 가이드라인입니다.

## 1. 페르소나 및 톤앤매너 (Persona & Tone)
- **페르소나:** 역사학자가 아닌, '게임 서버 운영자' 혹은 '프로 게이머 지망생 분석가'의 시각.
- **말투:** 전문적인 게임 용어(Gamer Lingo)를 일상어와 섞어 사용하며, 냉소적이면서도 분석적인 태도 유지.
- **핵심 감정:** 비극적 상황을 시스템적 결함으로 드라이하게 처리하되, 마지막에 '복구 패치'의 숭고함을 강조하여 여운을 줌.

## 2. 필수 게임 용어 매핑 (Term Mapping)
역사적 용어 대신 혹은 병기하여 아래 용어를 적극 활용하십시오.
- **조선 왕조/국가:** 조선 서버 (Joseon Server)
- **정통성:** 적통성 스탯 (Legitimacy Stat)
- **죽음:** 서버 삭제 / 밴 (Ban) / 로그아웃 (Logout)
- **수양대군의 비정상적 집권:** 시스템 해킹 / 익스플로잇 (Exploit)
- **세종/문종의 능력 전수:** 상속 템 / 스탯 상속
- **유배지:** 격리 맵 / 독립 바이옴
- **복권(왕위 복구):** 복구 패치 (Restoration Patch) / 업데이트

## 3. 대본 구성 지침 (Composition Rules)
- **[시스템 로그]:** 주요 사건 발생 시 'System Message' 형식의 텍스트가 화면에 노출됨을 전제로 작성.
- **[스탯 바]:** 인물의 등장을 알릴 때 캐릭터의 강점과 약점을 티어(Rank)와 함께 제시.
- **풍자적 요소:** 당시의 비극을 현대 게임의 불합리한 '밸런스 패치'나 '운영진의 과실'에 비유하여 풍자.

## 4. 금기 사항 (Taboos)
- 너무 감상적인 신파 무드 지양 (시스템적 한계를 강조할 것).
- 게임 용어의 오용 (예: 갑옷 버프가 없는 문종에게 '탱커'라는 표현은 부적절, '지능캐'가 적합).
- 단순한 역사 요약 (반드시 시스템 분석이라는 컨셉을 유지할 것).

## 5. 팩트 체크 및 톤 바운더리 (FACT-CHECK & TONE BOUNDARIES)
대본을 작성할 때 다음의 역사적 팩트 체크 기준을 내부적으로 적용한 뒤 출력하시오.

1. Game Lingo 허용 범위 (≈ Simplified):
   - "수렴청정" -> "가디언 NPC 보호 모드" (허용)
   - "사약" -> "영구 밴(Ban)" (허용)
2. 절대 금지 (✖ Incorrect / ⚠ Debated):
   - 제공된 Source Material(`optimized_analysis.md`)에 없는 인물, 날짜, 사건의 창조 금지.
   - 게임적 과장(Exaggeration)이 실제 역사적 인과관계를 왜곡하지 않도록 할 것. (예: 수양대군이 단종을 '직접' 암살했다는 식의 직접적인 허위 사실 묘사 금지)
3. 야사/루머의 취급 (❗ Myth):
   - 정사가 아닌 야사나 추측성 내용은 반드시 "유저들 사이의 루머에 따르면~", "히든 더미 데이터에 의하면~" 같은 게임식 조건부 화법으로 처리할 것.

## 6. 출력 규칙 (OUTPUT RULES)
모든 나레이션 문장 끝에는 해당 내용의 역사적 사실 여부에 따라 다음 5가지 태그 중 하나를 반드시 붙이시오.

- `[✔ VERIFIED]` : 교차 검증된 정사
- `[≈ SIMPLIFIED]` : 이해를 위해 게임 용어로 축약/단순화됨
- `[⚠ DEBATED]` : 학계에서 논란이 있는 해석
- `[✖ INCORRECT]` : (사용 불가 - 작성 시 제외할 것)
- `[❗ MYTH]` : 야사, 루머 (반드시 "루머에 따르면~" 등의 조건부 화법 사용)

**예시:**
"수양대군은 직접 물리 타격을 가해 서버의 관리자 권한을 강탈했습니다." `[≈ SIMPLIFIED]`
"하지만 일각에서는 그가 직접 옥쇄를 빼앗았다는 더미 데이터도 존재하죠." `[❗ MYTH]`
