**\# LLM Wiki \- 개인 지식 베이스**

\> Andrej Karpathy의 아이디어: "지식 베이스 유지의 지루한 부분은 정리 작업이다. LLM은 인간이 포기하는 교차 참조와 일관성 유지를 잘 해낸다."

**\---**

**\#\# 이 시스템이 하는 일**

AI와 대화하면 대화가 끝나면 다 사라집니다.  
이 시스템은 **\*\*대화 속 지식을 파일로 저장\*\***해서 다음 대화에서 이어갑니다.

\`\`\`  
당신 \= 편집장 (뭘 넣을지 결정)  
AI   \= 기자 (정리, 연결, 업데이트)  
\`\`\`

자료를 넣을수록 위키가 두꺼워지고, AI가 참고할 맥락이 늘어납니다.  
**\*\*지식이 복리로 성장합니다.\*\***

**\---**

**\#\# 폴더 구조**

도메인(관심 분야)별로 폴더를 나눈다. 아래는 예시:

\`\`\`  
내-위키/  
 .claude/  
   CLAUDE.md         ← 이 파일  
 sources/  
   cooking/          ← 원본 자료 (AI가 절대 수정 안 함)  
   business/  
 wiki/  
   cooking/          ← 정리된 지식 페이지  
   business/  
   index.md          ← 전체 목차  
   log.md            ← 작업 기록  
\`\`\`

도메인 예시 (본인에 맞게 바꾸세요):  
\- 요리 블로거: \`cooking/\`, \`ingredient/\`, \`content/\`  
\- 개발자: \`tech/\`, \`project/\`, \`career/\`  
\- 투자자: \`economy/\`, \`stock/\`, \`insight/\`  
\- 유튜버: \`writing/\`, \`production/\`, \`growth/\`

**\#\# 세션 시작**

새 대화가 시작되면 자동으로:  
1\. \`wiki/index.md\` 읽기 → 현재 위키 상태 파악  
2\. \`wiki/log.md\` 마지막 20줄 읽기 → 최근 작업 맥락 파악  
3\. 한 줄 현황 보고 \+ 명령어 리마인드

**\---**

**\#\# 핵심 작업 3가지**

**\#\#\# 수집 (Ingest)**  
자료를 넣으면 AI가 정리한다.  
1\. 자료를 읽고 핵심을 파악  
2\. \`sources/\`에 원본 저장  
3\. \`wiki/\`에 정리된 페이지 생성  
4\. \`wiki/index.md\`에 목차 등록  
5\. \`wiki/log.md\`에 작업 기록  
6\. 기존 페이지와 관련 있으면 **\*\*\[\[링크\]\]로 연결\*\***

**\#\#\# 질문 (Query)**  
질문하면 위키를 먼저 찾아보고 답한다.  
\- 위키에 답이 있으면 → 위키 기반 답변  
\- 새로운 분석이 나오면 → 위키 페이지로 저장

**\#\#\# 점검 (Lint)**  
위키가 커지면 건강 상태를 점검한다.  
\- 모순되는 내용  
\- 오래된 정보  
\- 연결 안 된 고립 페이지

**\---**

**\#\# 뭘 넣을 수 있나요?**

거의 뭐든 됩니다:

| 종류 | 넣는 방법 | 예시 |  
|------|----------|------|  
| 웹 글/블로그 | URL을 던진다 | "이거 수집해줘 https://..." |  
| 유튜브 영상 | URL을 던진다 | "이 영상 수집해줘 https://youtube..." |  
| 책 내용 | 텍스트를 붙여넣는다 | "이 책 내용 수집해줘: ..." |  
| 내 아이디어 | 말로 설명한다 | "이런 생각이 들었는데 기록해줘: ..." |  
| PDF/논문 | 파일을 sources/에 넣고 알린다 | "sources/에 넣은 파일 정리해줘" |  
| 회의 메모 | 텍스트를 붙여넣는다 | "오늘 회의 내용 정리해줘: ..." |

**\---**

**\#\# 명령어**

| 키워드 | 뜻 | 예시 |  
|--------|------|------|  
| \`수집\` | 자료 넣기 | "이거 위키에 넣어줘", URL 던지기 |  
| \`질문\` | 물어보기 | "\~가 뭐야?", "\~에 대해 정리해줘" |  
| \`점검\` | 상태 확인 | "위키 상태 어때?" |  
| \`현황\` | 보고 | "지금 어디까지?" |

**\#\#\# 연결 (이 시스템의 핵심 가치)**  
\- \`\[\[페이지명\]\]\`으로 페이지끼리 연결한다  
\- **\*\*AI의 핵심 역할\*\***: 새 자료가 들어오면 기존 페이지와의 연결을 자동으로 찾아서 만든다  
\- 이 연결이 쌓이면 지식 그래프가 형성된다  
\- Obsidian으로 열면 그래프 뷰에서 연결을 시각적으로 볼 수 있다

**\#\#\# 페이지 크기 가이드**  
\- 200줄 이하: 정상  
\- 200\~400줄: 좀 길지만 괜찮음  
\- 400줄 초과: **\*\*분할 필요\*\*** → 허브 페이지(개요+링크) \+ 세부 페이지로 나눈다

**\---**

**\#\# sources/ 규칙**

원본 자료는 **\*\*불변\*\***. AI는 읽기만 하고 절대 수정하지 않는다.

파일명: \`날짜-주제-제목.md\`  
\- 예: \`sources/cooking/2026-04-16-fermentation-kimchi-science.md\`

\`\`\`markdown  
**\---**  
title: 원본 제목  
source: URL 또는 출처  
date: 2026-04-16  
type: article / book / video  
**\---**

(원본 내용)  
\`\`\`

**\#\# 토큰 절약**

위키가 100페이지를 넘어도 AI가 전부 읽지 않는다:  
1\. \`index.md\` 목차만 먼저 읽기  
2\. 필요한 페이지 2\~3개만 골라 읽기  
3\. 한 세션에서 같은 페이지 두 번 읽지 않기

비유: 도서관에서 책 전부를 읽는 게 아니라, 목차 보고 필요한 챕터만 펴는 것.

**\---**

**\#\# 마찰 감지 (자동 안전장치)**

작업하다가 아래 상황이 생기면 AI가 사용자에게 알린다:  
\- 새 자료가 어느 도메인에 넣을지 애매함 → "도메인 추가할까요?"  
\- 한 도메인에 페이지가 너무 몰림 → "하위 분류를 만들까요?"  
\- 새 페이지가 기존 페이지와 많이 겹침 → "합칠까요?"  
\- 두 페이지가 서로 다른 말을 함 → "어느 쪽이 맞나요?"

이건 AI가 알아서 하니까 신경 안 써도 됩니다.

**\#\# 어떻게 작동하는가**

\`\`\`  
\[세션 1\] 자료 3개 수집 → wiki 페이지 3개 생성 → 파일 저장  
   ↓ (대화 종료 — AI 기억 사라짐)  
\[세션 2\] CLAUDE.md 자동 로딩 → index.md 읽기 → "아, 3페이지 있구나"  
   ↓ (새 자료 2개 수집 → 기존 페이지와 연결)  
\[세션 3\] 5페이지 상태에서 질문 → AI가 위키를 참고해서 더 정확한 답변  
   ↓  
\[세션 10\] 30페이지 → 질문하면 여러 페이지를 종합한 깊은 답변  
\`\`\`

**\*\*핵심\*\***: AI의 대화 기억은 매번 사라지지만, 파일은 남는다.  
CLAUDE.md가 "어떻게 행동할지", wiki/가 "뭘 알고 있는지"를 매번 알려준다.

**\#\# 시작하기**

1\. 폴더를 만든다 (위 구조 참고)  
2\. 이 파일을 \`.claude/CLAUDE.md\`에 넣는다  
3\. \`wiki/index.md\`와 \`wiki/log.md\`를 만든다 (위 템플릿 복사)  
4\. Claude Code로 이 폴더를 연다  
5\. 첫 자료를 던진다: "이거 수집해줘" \+ URL

끝. 나머지는 AI가 알아서 합니다.

명령어: \`수집\` · \`질문\` · \`점검\` · \`현황\`
