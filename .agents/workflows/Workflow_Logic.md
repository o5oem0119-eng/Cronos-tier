# 🛠 Chronos Engine: Script Generation Skill Logic

본 문서는 성공 사례를 분석하여 [역사+게임] 융합 대본을 생성하는 7단계 표준 공정을 정의한다.

---

### Phase 1. 성공 요인 분석 (Success Analysis)
- **Action**: 입력된 레퍼런스 대본의 고성과 요인 분석.
- **Output**: [구조적 특징, 후킹 포인트, 정보 밀도, 게임적 전개 방식] 리스트.
- **Reference**: Tier Zoo 스타일의 '성능 분석' 관점을 적용할 것.

### Phase 2. 팩트 체크 (Fact Checking)
- **Action**: 대본 내 역사적 사실을 검증. (필요 시 web_search 도구 활용)
- **Output**: `History_Fact_Sheet.md`의 1번 항목(Timeline) 업데이트.

### Phase 3. 추가 정보 검색 (Deep Research)
- **Action**: 보강이 필요한 역사적 비하인드 및 시각적 데이터 수집.
- **Output**: `History_Fact_Sheet.md`의 2, 3번 항목(Character, Visual) 업데이트.

### Phase 4. 대본 초안 작성 (Draft Scripting)
- **Action**: 위 단계의 데이터를 바탕으로 약 10분 분량의 대본 작성.
- **Tone & Manner**: 게임 캐스터 스타일, 역사 용어의 게임화 (예: 유배 -> 서버 추방).
- **Format**: 나레이션과 자막(UI)이 구분된 마크다운 형식.

### Phase 5. 검토 및 수정 (Review & Refinement)
- **Action**: 자체 피드백 루프 수행.
- **Checklist**: 지루한 구간 유무, 팩트의 왜곡 여부, 게임적 비유의 적절성.

### Phase 6. 스토리보드 제작 (Visual Storyboard)
- **Action**: 씬별 비주얼 연출 가이드 생성.
- **Remotion Connect**: 필요한 UI 컴포넌트(HP 바, 인벤토리 창 등) 명시.

### Phase 7. 메타데이터 생성 (Upload Package)
- **Action**: SEO 최적화 정보 생성.
- **Deliverables**: 고클릭 제목(3안), 영상 설명란, 핵심 태그, 썸네일 텍스트.

---
*주의: 모든 결과물은 한국어로 출력하며, 역사적 엄밀성과 게임적 재미의 균형을 5:5로 유지한다.*
