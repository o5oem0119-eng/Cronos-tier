# ⚙️ Chronos Engine: Production & UI Logic

본 문서는 대본(Script)을 바탕으로 영상 콘텐츠를 자동 조립할 때 적용되는 기술적 연출 규정, UI 상호작용, 장면 전환 및 사운드 동기화 로직을 정의한다.

---

## 1. 이원화 연출 시스템 (Dual-Layer Interface)
나레이션(해설)과 UI(시스템 로그)의 역할을 명확히 분리한다.

* **Layer 1: 나레이션 (Voice-Over/Subtitle)**
  - 역할: 역사적 사건의 배경 설명 및 TierZoo식 전략 분석.
  - 규칙: UI 대사창 문구를 그대로 읽지 않음. 텍스트가 '결과'라면, 나레이션은 '과정'과 '이유'를 설명.
  
* **Layer 2: 시스템 로그 (Pokemon Style Textbox)**
  - 역할: 사건의 타격 판정, 스탯 변화, 핵심 대사 출력.
  - 비주얼: `image_cde136.png` 스타일의 하단 픽셀 대사창 활용.

---

## 2. 장면 전환 규칙 (Scene Transition Rules)
프로그램은 팩트 시트(History_Fact_Sheet)의 사건 단위를 기준으로 화면을 전환한다.

* **사건 전환 (Event Change):** 팩트 시트의 항목이 바뀌면(예: 문종 사망 -> 김종서 기습), 이전 장면을 **0.5초간 페이드 아웃(Fade-out)** 후 다음 이미지를 노출한다.
* **긴급 상황 (Critical Event):** 계유정난 기습 등 급격한 변화 시에는 페이드 대신 **화이트 아웃(White-out) 또는 글리치(Glitch)** 효과를 사용하여 충격을 표현한다.

## 3. 사운드 트리거 로직 (Sound Trigger Logic)
특정 UI 이벤트가 발생하는 프레임에 맞춰 효과음을 자동 삽입한다.

| 발생 이벤트 | 출력 효과음 (SFX) | 연출 효과 |
| :--- | :--- | :--- |
| **텍스트 출력** | `text_scroll.mp3` | 포켓몬식 또르르 소리 (텍스트 길이에 비례) |
| **치명적 타격** | `critical_hit.mp3` | '효과는 굉장했다' 텍스트와 동시에 재생 |
| **HP/스탯 하락** | `stat_down.mp3` | 게이지가 줄어드는 애니메이션과 동기화 |
| **인물 리타이어** | `faint.mp3` | 캐릭터가 화면에서 사라지는 순간 재생 |
| **장면 전환** | `swoosh.mp3` | 화면이 바뀔 때 부드러운 전환음 삽입 |

## 4. UI 컴포넌트 데이터 매핑 (Remotion Mapping)

### 4.1 상단 상태창 (Status Bar)
* **단종(Player):** 화면 좌측 상단 배치. 이름, Lv(나이), HP(정통성) 바 노출.
* **상대(Target):** 화면 우측 하단 배치. 이름, 위험도(Threat Level) 바 노출.

### 4.2 하단 메시지 창 (Message Box)
* **영역:** 화면 하단 30% 고정 배정. 
* **폰트:** 픽셀 도트 폰트 사용.
* **동기화:** 나레이션이 끝나는 지점에 맞춰 다음 시스템 메시지로 자동 전환.
